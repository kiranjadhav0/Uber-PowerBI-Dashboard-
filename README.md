# Uber Ride Booking Data Analysis

## Project Overview

This project analyzes Uber ride booking data to understand customer behavior, ride completion trends, cancellations, revenue generation, and operational performance. The analysis was conducted using Python, SQL, and Power BI as part of a data analytics case study.

The goal of this project is to identify business insights and provide data-driven recommendations that can improve ride completion rates, customer retention, and overall operational efficiency.

---

## Business Problem

Ride-sharing platforms often face challenges such as ride cancellations, booking failures, driver availability issues, and customer retention. Understanding these factors is essential for improving customer satisfaction and business performance.

This project aims to answer the following questions:

* What is the ride completion rate?
* What causes ride cancellations?
* Which customer segments generate the most revenue?
* How effectively does Uber retain customers?
* What operational improvements can be recommended?

---

## Dataset Information

* Dataset: Uber Ride Booking Dataset
* Records: Approximately 150,000 bookings
* Type: Public Dataset
* Format: CSV

The dataset contains booking information, ride status, vehicle details, customer activity, and revenue-related data.

## Key Findings

* Ride Completion Rate: Approximately 60–62%
* Ride Cancellation Rate: Approximately 25%
* Failed Booking Rate: Approximately 13%
* Repeat customers generate most of the revenue.
* Customer retention drops significantly after the first ride.
* Driver availability issues contribute to booking failures during peak demand periods.

---

## Business Recommendations

* Improve driver allocation during peak hours.
* Reduce ride cancellations through operational improvements.
* Introduce customer loyalty programs.
* Focus on high-revenue routes.
* Continuously monitor key performance indicators using dashboards.
